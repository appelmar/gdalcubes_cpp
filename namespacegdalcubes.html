Detailed Description

Simplistic library for datetime objects. The assumption of the library is that durations are only meaningful if given as a single number with a single datetime unit. Whether or not this assumption is valid depends on the application.

GET /version POST /file (name query, body file) POST /cube (json process descr), return cube_id GET /cube/{cube_id} POST /cube/{cube_id}/{chunk_id}/start GET /cube/{cube_id}/{chunk_id}/status status= "notrequested", "submitted" "queued" "running" "canceled" "finished" "error" GET /cube/{cube_id}/{chunk_id}/download
